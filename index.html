أبني أنظمة هندسية متينة لا مجرد أكواد

أؤمن بأن البرمجة هي هندسة وحل للمشاكل. انتقلت من مرحلة "المهم أن يعمل الكود" إلى بناء أنظمة Microservices قابلة للتوسع، مؤمنة بعمق، وتعتمد على أنماط التصميم (Design Patterns) المعمارية باستخدام Laravel و React.

تصفح الأنظمة تحميل السيرة الذاتية
// Architecting a scalable system
class SoftwareEngineer {
  public $name = "Abdalrhman Hmadi";
  public $motto = "Read Logs, Don't Guess";

  public function buildSystem() {
    return new MicroserviceArchitecture([
      'Backend' => 'Laravel / RESTful APIs',
      'Frontend' => 'React SPA',
      'Database' => 'MongoDB / Relational',
      'AI' => 'Python / FastAPI'
    ]);
  }
}

التحديات المعمارية والحلول:

  • تكامل Laravel مع MongoDB: تجاوزت مشاكل التخزين المؤقت (Cache) وضبط النظام ليقرأ الرابط السحابي بكفاءة.
  • هندسة المصادقة (JWT): حللت انهيار الخادم الناجم عن اختلاف المعرفات من خلال تتبع دقيق للسجلات وإعادة بناء نموذج البيانات.
  • نشر خدمة الذكاء الاصطناعي (Deployment): دمجت الكود المصدري لمكتبة Python/FastAPI خاصة بشكل مستقل داخل المشروع.

تطبيق عملي لمفاهيم هندسة البرمجيات لبناء نظام آمن، قابل للتوسع، وسهل الصيانة.

أبرز التطبيقات الهندسية:

  • إدارة بوابات الدفع بمرونة: استبدلت الشروط المعقدة بتطبيق نمط التصميم Strategy Pattern.
  • الأمان الدقيق (Granular Permissions): تصميم نظام صلاحيات عميق يفصل بين أدوار المدراء.
  • فصل المهام (Event-Driven Architecture): استخدام الـ Events و Listeners لمعالجة التحديثات فورياً.
